Install

  1. Download the DMG above and drag AmphetamineXL to Applications
  2. Clear the quarantine flag (macOS blocks unsigned apps):
xattr -cr /Applications/AmphetamineXL.app
  1. Open AmphetamineXL — ⚡ appears in your menu bar
  2. Required one-time setup — disable Apple Silicon deep standby:
sudo pmset -a standby 0 && sudo pmset -a hibernatemode 0 && sudo pmset -a autopoweroff 0
  1. Close your lid — Mac stays awake ✅

Verify it's working

Run these to confirm everything is active:

pgrep -x AmphetamineXL && echo "✅ App running" || echo "❌ Not running"
pmset -g assertions | grep -E "AmphetamineXL|caffeinate|UserIsActive"
pmset -g | grep -E "standby|hibernatemode|autopoweroff"

Debugging

View the app's detailed logs:

log show --predicate 'subsystem == "com.hannojacobs.AmphetamineXL"' --last 10m

Check sleep/wake history:

pmset -g log | grep -E "Sleep|Wake|Clamshell" | tail -20

Revert standby settings (re-enable deep sleep):

sudo pmset -a standby 1 && sudo pmset -a hibernatemode 3 && sudo pmset -a autopoweroff 1

How it works

On Apple Silicon Macs, closing the lid triggers a hardware-level "clamshell sleep" controlled by the SMC chip. This ignores all software sleep prevention — IOKit assertions, caffeinate, pmset, everything.

The only thing the SMC respects is HID activity (Human Interface Device — mouse, keyboard, etc). AmphetamineXL posts synthetic mouse events via CGEventCreateMouseEvent every second. The SMC sees real input activity and won't enter clamshell sleep.

This technique was reverse-engineered from Amphetamine's binary — they call it "Drive Alive".
